Declaration Management System

My second year at Boltrics was spent developing the new Declaration Management System. The declaration management system provides a way to view and create customs declaration documents and a way to communicate with customs. The new declaration management system (DMS) came as a replacement for the current one (AGS). Because the definitions of the messages changed significantly, and because the old AGS code had become spaghetti, the decision was made to create a new module from scratch.

The new declaration management system (DMS) is based on the Message Implementation Guide provided by customs. In this guide you can find which elements have to be present in the declaration messages sent to customs. Per element it is given whether it is always required, optional, not used or whether it depends on some other conditions. Each of these conditions were provided in another guide.

One of the main objectives of the new declaration management system was to provide the user with enough information to be able to solve problems with their declaration themselves. The old system did not provide much information on what went wrong, if any. Because of this, all validations provided by customs were implemented and their results were logged such that it became visible what was wrong.

Another focuspoint was integration with the Warehouse Management System and Transport Management System. It was possible to create customs documents based on those documents, so this had to be possible in the new declaration management system as well. Based on the factory design pattern, a new way of creating DMS documents was accomplished.
